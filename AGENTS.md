# Volunteer Hub — хакатон (дедлайн 20.07 09:00, UTC+5)

## Контекст (НЕ переспрашивать пользователя, работать автономно)

Хакатон «Startup Platform для партнёра», соло. Партнёр выбран: **Employee Volunteer Program (NVS & Zhurekten Zhurekke)** — программа корпоративного волонтёрства Chevron в Казахстане (субботники, посадка деревьев, поддержка детей с инвалидностью и из малообеспеченных семей; города: Астана, Алматы, Атырау).

**Продукт:** Volunteer Hub — веб-приложение, где координатор публикует волонтёрские события, а сотрудники записываются на них в 1 клик. Боль: сейчас записи на такие акции ведутся в рассылках/Excel, координатор не видит, кто и сколько записался.

**Критерии хакатона:** работающий продукт (40%), релевантность сфере партнёра (25%), фидбэк 5+ реальных пользователей (20%), питч (15%).

## Стек (ничего не добавлять!)

- Один статический сайт: `index.html` + `app.js` + `config.js`. Без фреймворков, без сборки, без npm-пакетов в рантайме.
- Supabase через CDN: `<script src="https://cdn.jsdelivr.net/npm/@supabase/supabase-js@2"></script>` (глобальный `window.supabase`).
- БД: Supabase (схема в `schema.sql`, пользователь уже применил её и вставил ключи в `config.js`).
- Деплой: Vercel, статика, `npx vercel --prod --yes`.
- UI на русском. Стили — один `<style>` в index.html, минимально прилично.

## Функционал (всё, больше ничего)

1. Список событий из таблицы `events`: название, город, дата, категория, описание, сколько записалось (`signups`), кнопка «Записаться».
2. Запись на событие: имя + контакт → insert в `signups`. После записи обновить список.
3. «Я координатор»: кнопка открывает форму добавления события; поле «код организатора», сверка со строкой `zhurek2026` в app.js. Неверный код — alert, не сохранять. (ponytail: код косметический, спам-защита не нужна на хакатон)
4. Форма «Оставить отзыв о платформе»: имя/роль, оценка 1–5, текст → insert в `feedback`. Это сбор фидбэка для хакатона.
